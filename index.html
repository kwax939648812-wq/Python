跟著我們的小貓頭鷹，一起學習版本控制的魔法吧！

git add

將檔案變更加入暫存區 (Staging Area)，準備進行提交。想像成把衣服放進洗衣籃。

git commit

將暫存區的變更永久記錄到本地倉庫，並附上說明。就像寫日記一樣。

git push

將本地倉庫的變更推送到遠端倉庫，與團隊分享成果。把日記分享給朋友看！

git pull

從遠端倉庫獲取最新變更並合併到本地分支。看看朋友今天寫了什麼日記。

git revert

建立一個新的提交來撤銷指定的提交，保留歷史記錄。後悔時的安全選擇。

git checkout

切換到不同的分支或特定的提交版本。快速換一套衣服或回到過去的時間點。

git merge

將一個分支的變更合併到另一個分支。把兩本日記合併成一本。
